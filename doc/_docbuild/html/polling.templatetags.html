polling.templatetags.reusable_tags.check_exist(search, datas)[source]¶

check string in list such ((x,y),(a,b)) then show result on match.

Parameters:
  • search – a key value to search , the first index
  • datas – a list of data

assume datas has ((‘a’,b’),(‘c’,’d’)). then when user do in template

{{ "a" | check_exist }}

b will be shown when exist, or return nothing when couldn’t find a match

polling.templatetags.reusable_tags.get_range(value)[source]¶

Filter - returns a list containing range made from given value created by zalun

usage (in template):

<ul>{% for i in 3|get_range %}
  <li>{{ i }}. Do something</li>
{% endfor %}</ul>

Results with the HTML:

<ul>
  <li>0. Do something</li>
  <li>1. Do something</li>
  <li>2. Do something</li>
</ul>

Instead of 3, any number may use the variable set in the views (need to change the source)

polling.templatetags.reusable_tags.if_exist(var, output)[source]¶

if value is true return output, this is the opposite of default tag in django.

Parameters:
  • var – common variables
  • output – string output

assume context sent a variable name check, usage(in template):

{{ check | if_exist:"show me something" }}

Results : When chek is True show show me something and when False returns an empty string
